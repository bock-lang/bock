# Spec Changes — 2026-03-21 11:00

Addition of §17.7 — Codegen Rule Learning model.

---

## Context

Build testing of the Tier 2 (rule-based) code generators
revealed an unbounded category of target-specific idiom
gaps — prelude function mappings (println → console.log),
syntactic conventions (multi-line strings, macro invocation),
and per-version API differences. Manual enumeration doesn't
scale.

Design discussion evaluated a feedback loop where the LLM
fixes compilation failures AND extracts the rule that would
have prevented them. The rule-based tier grows organically.

## Decision

Add §17.7 as a post-v1 design note. The feedback loop is
part of the language's transpilation architecture but is not
on the critical path for the initial 61-package implementation.

Key design choices made:
- **Local-first architecture.** No centralized build server.
  Rules distributed as packages through existing registry.
- **Scope boundary.** Loop handles prelude mapping and
  syntactic idioms. Semantic gap bridging and runtime
  synthesis stay in AI tier. Promotion path exists for
  stabilized AI patterns.
- **Strictness alignment.** Follows existing graduated model:
  auto in sketch, logged in development, pinned in production.
- **Decision manifest integration.** Rules logged as manifest
  entries, compatible with aura inspect/override.
- **Rule format TBD.** Intentionally deferred pending Tier 2
  implementation experience.

## Spec Amendment

### §17.7 — Codegen Rule Learning (Post-v1) — NEW
Inserted between §17.6 (Capability Gap Resolution) and §18
(Standard Library). Covers:
- The feedback loop model
- Rule scope (prelude mapping, syntactic idioms)
- Scope exclusions (semantic gaps, runtime synthesis)
- Rule format (TBD, declarative pattern-template)
- Strictness interaction table
- Decision manifest integration
- Distribution via package registry

No other sections modified.

## Implementation Implications

No changes to the current 61-package plan. This section
establishes design intent for a future phase. When the
feedback loop is implemented, it will likely be:
- A P9.x extension or new Phase 11
- 3-4 packages: rule format, extraction pipeline,
  validation framework, CLI integration
- Dependent on Phase 9 (codegen) and Phase 10 (build)
  being complete
