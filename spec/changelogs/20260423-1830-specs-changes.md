# Spec Changes — 2026-04-20 18:30 UTC

Styling adjustment: **BOCK** → **Bock** in prose and proper-noun
contexts. Lowercase `bock` (CLI binary, crate prefix, file
extension, config files, module paths) is unchanged.

---

## Rationale

All-caps `BOCK` reads as an acronym and implicitly demands
expansion every time a reader encounters it. Title-case `Bock`
reads as a proper noun — which is what the language name
actually is. The backronym (*Bound Objective Context Kit*)
exists only for branding/marketing use; it does not appear in
the spec or playbook at all, so styling the name as title-case
removes unnecessary acronym signaling throughout the technical
documentation.

The lowercase form `bock` is semantically different — it's the
CLI binary, the crate prefix, the file extension. Those don't
change and shouldn't; they follow the Unix/Rust convention of
lowercase identifiers for tools and libraries.
