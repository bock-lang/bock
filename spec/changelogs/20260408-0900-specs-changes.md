# Spec Changes — 2026-04-08 09:00

Addition of `core.time` module with `Duration`, `Instant`, `sleep`,
and the `Clock` effect.

---

## Context

Implementation and async testing surfaced the need for sleep,
duration arithmetic, and monotonic time primitives. Neither the
spec nor stdlib provided these. The implementation chat proposed
`core.time`; the design chat reviewed the proposal and made six
refinements before adoption:

1. **Clock effect ownership clarified.** The effect lives in
   `core.time`, not `std.time`. `std.time` provides the default
   handler and layers wall-clock operations on top.
2. **Duration representation pinned.** Internally `Int64` nanos
   (~±292 year range). No per-target variance.
3. **Methods expanded.** Added `zero()`, `is_zero()`,
   `is_negative()`, `abs()` — essentials missing from the proposal.
4. **Scalar arithmetic added.** `Duration * Int` and `Duration / Int`
   for backoff algorithms and similar patterns.
5. **`sleep` in the prelude.** Used often enough that import
   friction would be annoying.
6. **`-` operator as sugar for `duration_since`.** Method is
   canonical. Added `Instant + Duration` and `Instant - Duration`
   which the proposal omitted.

---

## Spec Amendments

### §18.2 — Prelude — EXPANDED
Added `Duration`, `Instant` to always-available types.
Added `sleep` to always-available utility functions.

### §18.3 — Core Modules — EXPANDED
Added entry:
`core.time` — `Duration`, `Instant`, `sleep`, monotonic time
primitives, `Clock` effect.

### §18.3.1 — core.time — NEW SUBSECTION
Defines:
- `Duration` type (Int64 nanos, ±292 year range)
- `Instant` type (monotonic, per-process)
- `sleep(Duration) -> Void with Clock` prelude function
- Duration constructors: `zero`, `nanos`, `micros`, `millis`,
  `seconds`, `minutes`, `hours`
- Duration methods: `as_nanos`, `as_millis`, `as_seconds`,
  `is_zero`, `is_negative`, `abs`
- Duration arithmetic: `+`, `-`, `* Int`, `/ Int`
- Instant operations: `now() with Clock`, `elapsed() with Clock`,
  `duration_since(earlier)`
- Instant arithmetic: `Instant - Instant = Duration`,
  `Instant ± Duration = Instant`
- Clock effect: `now_monotonic()`, `sleep(Duration)`
- MockClock virtual time advancement for fast tests

### §18.4 — Standard Modules — UPDATED
`std.time` description changed from:
> "Timestamps, durations, dates, time zones, `Clock` effect."
to:
> "Wall-clock time, dates, timezones, calendar arithmetic.
> Extends `core.time` with the default `Clock` handler
> (`SystemClock`)."

This clarifies the ownership boundary: `core.time` owns the
Clock effect and monotonic primitives; `std.time` provides the
default handler and wall-clock operations.

## Implementation Implications

Affected packages:

**P5.6 (Standard value operations)** — The `Duration` and
`Instant` types should be registered as built-in value types
alongside the other primitives. `sleep` should be registered
as a builtin function.

**P6.1 (Primitive types)** — Add `Duration` and `Instant` with
method suites. Register arithmetic operator overloads.

**P6.5 (Core traits)** — Ensure `Duration` and `Instant` implement
`Comparable` and `Equatable` (required for the `<`/`>`/`==`
operators).

If these packages have already been run, this becomes a cleanup
task. If not, the updated s14-stdlib.md will guide them correctly.
