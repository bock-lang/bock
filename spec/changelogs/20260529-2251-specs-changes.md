# Core-spec reconciliation — §11.4 performance literals, §13.3/§13.4 Reserved, §18.3 core-module v1/v1.x tiering

**Date:** 2026-05-29
**Affects:** §11.4, §13.3, §13.4, §18.3 (and editorial cross-ref fix in `20260514-0449-specs-changes.md`)
**Type:** clarification (Design-chat core-spec decisions, 2026-05-29; reconciliation to v1 reality)

## Change

Three Design-chat decisions from 2026-05-29 are landed in the core spec.
None introduce new runtime behavior; they pin down literal syntax that
the checker already enforces (Q3), mark the concurrency surface that v1
does not ship (Q2), and tier the 15 `core.*` modules into v1 vs.
Reserved-for-v1.x with an explicit minimum-useful principle (Q1).

### Q3 — §11.4 `@performance` literal syntax

A normative **Literal syntax** paragraph is added to §11.4 after the
`100.ms` / `50.mb` example:

- `@performance` budgets require **unit-suffixed numeric literals**.
- Time budgets use `.ns`, `.us`, `.ms`, `.s`, `.min`, `.h`.
- Memory budgets use `.b`, `.kb`, `.mb`, `.gb`, `.tb`.
- Bare integers produce **E8003**.
- The unit suffix is part of the literal, not a method call
  (`100.ms` is a literal, not `100 . ms()`).
- Time units match the `Duration` constructor names in §18.3.1; memory
  units use **decimal** scaling (1 kb = 1000 b).

The exerciser `examples/spec-exercisers/context-audit/src/main.bock`
demonstrated the old bare-integer form
(`@performance(max_latency: 100, max_memory: 50)`) and is corrected to
`@performance(max_latency: 100.ms, max_memory: 50.mb)`; the leading
doc comment is updated to describe the unit-suffixed form rather than
the old `(ms)` / `(MB)` parenthetical. The example now `bock check`s
clean of E8003.

### Q2 — §13.3 Channels and §13.4 Synchronization Primitives → Reserved for v1.x

Both sections gain a leading **"Reserved for v1.x"** status note. The
existing design material is preserved as design intent:

- §13.3 keeps the `Channel[Message].new(buffer: 10)` send/receive
  example.
- §13.4 keeps the `Mutex` / `RwLock` / `Atomic` / `WaitGroup` /
  `OnceCell` enumeration.

Both bundle with `core.concurrency`, which is Reserved for v1.x per Q1.
This **completes the loop** the `20260514-0449-specs-changes.md`
cross-reference left open: that changelog asserted §13.3/§13.4 were
"Reserved per the D1+D2 batch", but no such batch changelog deferred
them — the leading notes are the missing piece. The 0449 cross-ref is
corrected (minimal edit) to point at the §13.3 / §13.4 leading notes
and this changelog, rather than a non-existent batch deferral. This is
not a factual-error correction of a wrong claim; the 0449 statement was
forward-looking and this changelog lands the deferral it anticipated.

### Q1 — §18.3 core-module v1/v1.x tiering

§18.3 is reframed so each of the 15 `core.*` modules is marked **v1** or
**Reserved for v1.x**, with a principle note: *a v1 module ships its
minimum-useful subset, not the full surface listed; acceptance for a v1
module = the conformance suite and a representative example compile and
run on every shipping target.* Per-module descriptions (including the
`String.len` Unicode-scalar-value rule) are preserved; §18.3.1
(core.time) is unchanged.

**v1 (11):** core.option, core.result, core.collections, core.string,
core.iter, core.compare, core.convert, core.error, core.effect,
core.time, core.test.

**Reserved for v1.x (4):** core.types (BigInt/Decimal), core.math
(constants/advanced fns), core.memory (Rc/Arc — bundles with FFI / §14),
core.concurrency (channels + sync primitives — Q2).

Deferred bits noted inline in v1 modules: collections defers `Deque`,
`SortedMap`, `Stack`, `Queue`, `BitSet`, `Array[T, N]`; string defers
`Regex`; test defers property and snapshot testing.

#### Q1 minimum-useful v1 surface per module (recorded by Design)

The spec body stays at the tiering altitude (module marker + the
deferred-bits note). The per-module method lists below are the
**minimum-useful v1 surface** as recorded by Design; they scope the
stdlib implementation and are reproduced here so they are captured even
though they are not normative spec prose. A v1 module may ship more than
this floor; it must not ship less.

- **core.option** — `Optional[T]` with `is_some`, `is_none`, `map`,
  `and_then` (flat_map), `or_else`, `unwrap_or`, `unwrap_or_else`,
  `ok_or` (→ `Result`), `filter`. Constructors `Some` / `None` via
  prelude.
- **core.result** — `Result[T, E]` with `is_ok`, `is_err`, `map`,
  `map_err`, `and_then`, `or_else`, `unwrap_or`, `unwrap_or_else`,
  `ok` (→ `Optional`). `?` operator support (language-level). `Ok` /
  `Err` via prelude.
- **core.collections** — `List[T]`: literal, `len`, `is_empty`, index
  get/set, `push`, `pop`, `insert`, `remove`, `contains`, `slice`,
  `iter`, `concat`. `Map[K, V]`: literal, `get` (→ `Optional`),
  `insert`, `remove`, `contains_key`, `keys`, `values`, `entries`,
  `len`. `Set[T]`: `insert`, `remove`, `contains`, `len`, `union`,
  `intersection`, `difference`, `iter`. `SortedSet[T]`: ordered
  iteration + the `Set` surface. (Deferred: Deque, SortedMap, Stack,
  Queue, BitSet, fixed-size `Array[T, N]`.)
- **core.string** — `len` (Unicode scalar count), `byte_len`,
  `is_empty`, `chars`, `bytes`, indexing by scalar, `slice`, `concat`
  / `+`, `contains`, `starts_with`, `ends_with`, `find`, `replace`,
  `split`, `trim` / `trim_start` / `trim_end`, `to_upper`, `to_lower`,
  `repeat`, `parse` (→ `Result`). `StringBuilder`: `new`, `append`,
  `append_char`, `build` / `to_string`, `len`. (Deferred: `Regex`.)
- **core.iter** — `Iterator` trait (`next` → `Optional`) plus
  combinators: `map`, `filter`, `filter_map`, `flat_map`, `take`,
  `skip`, `take_while`, `skip_while`, `enumerate`, `zip`, `chain`,
  `fold`, `reduce`, `for_each`, `count`, `sum`, `product`, `min`,
  `max`, `any`, `all`, `find`, `position`, `collect` (→ List/Map/Set).
- **core.compare** — `Ordering` (`Less`/`Equal`/`Greater`),
  `Comparable` (`compare` → `Ordering`; derives `<` `<=` `>` `>=`),
  `Equatable` (`eq`; derives `==` `!=`). Free helpers `min`, `max`,
  `clamp`.
- **core.convert** — `Into[T]` / `From[T]` (blanket From⇒Into),
  `TryFrom[T]` (→ `Result`), `Displayable` (`to_string`). Used by
  string interpolation and `parse`.
- **core.error** — `Error` base trait: `message` (→ String), optional
  `source` (→ `Optional[Error]`). Composes with `core.result` and the
  `?` operator; integrates with adaptive handlers (§10.8).
- **core.effect** — Effect primitives: `effect` declaration surface,
  `with <Handler>` handler installation, ambient-effect resolution,
  the `Clock` effect plumbing consumed by core.time. (Effect *erasure*
  remains Reserved per the prior changelog.)
- **core.time** — Per §18.3.1, unchanged: `Duration` (constructors
  `zero`/`nanos`/`micros`/`millis`/`seconds`/`minutes`/`hours`,
  methods `as_nanos`/`as_millis`/`as_seconds`/`is_zero`/`is_negative`/
  `abs`, arithmetic, `Comparable` + `Equatable`), `Instant`
  (`now`/`elapsed`/`duration_since` + arithmetic), `sleep`, the `Clock`
  effect.
- **core.test** — Assertions (`assert`, `assert_eq`, `assert_ne`,
  `assert_true`/`assert_false`, `assert_throws`/expected-error), BDD
  grouping (`describe` / `it` or equivalent), basic mocking
  (incl. `MockClock` for time), and benchmarking entry points.
  (Deferred: property-based testing, snapshot testing.)

The four Reserved-for-v1.x modules ship **no** v1 surface; their listed
types/functions are design intent:

- **core.types** — `BigInt`, `Decimal` (sized integer/float primitives
  are language-level, not part of this deferred surface).
- **core.math** — mathematical constants, advanced functions, numeric
  traits beyond the arithmetic that primitives already provide.
- **core.memory** — `Rc`, `Arc` for `@managed` contexts; bundles with
  FFI / §14.
- **core.concurrency** — `Channel`, `Mutex`, `RwLock`, `Atomic`,
  `WaitGroup`, `OnceCell` (the §13.3 / §13.4 surface, Q2).

## Rationale

These are the 2026-05-29 Design-chat core-spec decisions (Q1/Q2/Q3),
landed faithfully. Q3 documents syntax the checker already enforces
(E8003), so the spec now matches the implementation's contract. Q2 and
Q1 align the spec's normative surface with what v1 actually ships,
preserving the deferred surface as explicit design intent (consistent
with the §13.5 and §20 Reserved-marker pattern) so v1.x has clear
entry points rather than having to reconstruct intent. The minimum-
useful principle prevents the inverse failure mode — a v1 module
treated as "all-or-nothing" and slipping because its full surface is
large.

## Migration

- **`@performance`:** budgets must use unit-suffixed literals
  (`100.ms`, `50.mb`); bare integers were already rejected with E8003.
  No new rejection — this documents existing behavior.
- **Channels / sync primitives:** code using `Channel`, `Mutex`,
  `RwLock`, `Atomic`, `WaitGroup`, `OnceCell` targets v1.x; it is not
  part of the v1 surface.
- **core.\* modules:** the four Reserved modules (core.types,
  core.math, core.memory, core.concurrency) and the inline-deferred
  collection/string/test items are v1.x; v1 code should not depend on
  them.

## Implementation note (FOUND — spec ahead of checker on units)

The Q3 spec text declares the full time set `.ns/.us/.ms/.s/.min/.h`
and memory set `.b/.kb/.mb/.gb/.tb`. The current checker
(`compiler/crates/bock-air/src/context.rs`, `parse_duration` /
`parse_byte_size`) recognizes only `ns/us/ms/s` and `b/kb/mb/gb` —
`.min`, `.h`, and `.tb` are spec-declared but not yet accepted and
would currently emit E8003. The spec is the source of truth; this is a
v1 implementation follow-up (extend the two unit tables), not a spec
change. The landed example uses only `.ms` and `.mb`, which the checker
accepts today, so it checks clean. Filed here as FOUND for the impl
chat.
