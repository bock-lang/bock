# §1.1 identity amendment — conformance-tested semantic equivalence

**Date:** 2026-06-15
**Affects:** §1.1 (What Is Bock)
**Type:** addition (surfaces an existing, validated capability in the identity section)

## Change

§1.1 gains one sentence:

> Uniquely, Bock's multi-target output is conformance-tested for semantic
> equivalence: the same program is verified to behave identically on every
> target it ships to.

No new normative rule. The sentence surfaces in the identity section the
cross-target equivalence guarantee already operationalized by the conformance
execution suite (the cross-target testing tier; see the §13/§20 cross-target
testing and validation references) and already referenced later in the spec
("the cross-target semantic equivalence claim that distinguishes Bock from a
target-by-target source-to-source transpiler").

## Rationale

The 2026-06-09 strategic design audit (R1 — identity) and the operator-signed
marketing-positioning resolution (2026-06-15, returned from the marketing chat)
established this equivalence guarantee as Bock's lead identity. Design supplied
the truth basis; the operator signed the wording.

The verb is **"conformance-tested / verifies," not "proves."** Per the R9
validation ledger (`tracking/snapshot.md`), the suite *verifies* equivalence on a
*covered* surface (~850 fixture×target pairs, 0 failed ×5, with a standing
off-fixture caveat that dogfood-driven widening keeps load-bearing). "Proves"
would read as exhaustive formal proof and overclaim. The public marketing
identity sentence (marketing-owned, not part of this changelog) shares this verb
so the two stay aligned.

## Migration

None — documentation/identity clarification of an existing capability. No code,
grammar, or semantic change.
