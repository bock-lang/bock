# §17.2 / §15.2 / §10.8 / §17.4 CLI cross-references aligned to the §20.1 surface reconciled in #92

**Date:** 2026-05-30
**Affects:** §17.2, §15.2, §10.8, §17.4
**Type:** clarification (editorial cross-reference alignment, no normative change)

## Change

#92 reconciled §20.1 (the normative CLI surface) to v1 reality —
marking `bock build --optimize` / `--deliverable` / `--no-tests`,
`inspect --diff`, and `pin --all` as **Reserved for v1.x**, and
rewriting the `pin` / `override` flag forms to the actual v1 surface
(see `spec/changelogs/20260529-1905-section-20-cli-reconciliation.md`,
"Out-of-scope cross-references (follow-up sync)"). That reconciliation
was scoped to §20.1 / §20.7 / Appendix A. Four cross-references
elsewhere in the spec still cited the old, pre-reconciliation flag
forms. Each already pointed at §20.1 as the normative CLI surface, so
this is editorial consistency only — no design decision and no new
normative behavior.

The cross-references are realigned to match §20.1:

- **§17.2 (Tier 3 — AI Optimization)** — the Tier-3 activation
  reference to `bock build --optimize` now notes that the flag is
  **Reserved for v1.x** per §20.1 (it ships with the project-mode
  build work), matching §20.1's `bock build` flag list.

- **§15.2 (Test-runner-consumed annotations)** — the test-exclusion
  reference to `bock build --no-tests` now notes that the `--no-tests`
  opt-out is **Reserved for v1.x** per §20.1 (test inclusion is the v1
  default), matching §20.1's `bock build` flag list.

- **§10.8 (Adaptive Effect Handlers — Promotion path)** — the
  promotion command, previously written
  `bock override --promote <selection-id>`, is rewritten to the v1
  `override` surface per §20.1: `bock override <decision-id> --promote`
  (a bare `--promote` flag operating on the positional decision id).

- **§17.4 (Decision Manifest)** — the same
  `bock override --promote <selection-id>` form (the second occurrence;
  the upstream #92 changelog labeled the override cross-refs
  "§10.8 / §10.4", but §10.4 contains no such reference — the two real
  occurrences are §10.8 and §17.4) is rewritten identically to
  `bock override <decision-id> --promote`.

§20.1 itself is unchanged (it is the canonical CLI surface from #92).
The §20.6.2 description of the `--deliverable` / `--no-tests` *modes*
is also unchanged: it is inside §20, part of #92's reconciliation
scope, and is preserved there as the planned v1.x mode surface.

## Rationale

#92's reconciliation explicitly flagged these as out-of-scope
follow-up sync items, each cross-referencing §20.1 as the normative
CLI surface. Aligning the cross-references removes the remaining stale
citations of pre-reconciliation flag forms so the spec is internally
consistent with the §20.1 surface. This is a pure editorial
clarification: §20.1 is normative and already correct, and these
sections only cite it.

## Migration

None. No behavior changes. The `--optimize` / `--no-tests` flags were
never implemented in v1 (any script using them already failed), and
the `override --promote` rewrite describes the v1 form that already
shipped.
