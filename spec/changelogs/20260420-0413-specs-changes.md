# Specifications Changelog — 2026-04-20 04:13 UTC

## Summary

Language renamed from **Aura** to **BOCK** (expansion: *Bound Objective Context Kit*).

## Rationale

Codename "Aura" had blocking collisions (Salesforce Aura framework, UPenn AURA PL, Arch Linux Aura, esoteric Aura lang). BOCK selected after a 30+ candidate evaluation cycle against seven criteria (length, TLD availability, PL/framework/AI competition, non-tech mindspace, phonetic/visual confusability, story/solidity, unambiguous pronunciation).

Key validation findings for BOCK:

- Zero competing programming languages, compilers, frameworks, or AI products
- crates.io returns zero results for "bock" (critical for Rust-built compiler)
- `npm install bock` already owned (`doesdev/bock`)
- GitHub `doesdev/bock` already owned
- `.bock` file extension unclaimed
- No security/malware contamination, no CVE hits
- Unambiguous US English pronunciation (`/bɑːk/`)
- Clean Class 9 (software) trademark landscape; existing BOCK marks cluster in Class 7/11/32

Expansion *Bound Objective Context Kit* compresses both differentiators into four words:

- **Bound / Objective** → feature-declarative (developer declares intent; compiler binds it)
- **Context** → target-agnostic (same objective, many runtime contexts)
- **Kit** → honest framing of what ships (toolchain, not philosophy)

One-line pitch: *Declare your objective once, bind it to any context.*
