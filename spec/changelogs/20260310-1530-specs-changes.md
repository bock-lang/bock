# Spec & Implementation Changes — 2026-03-10 15:30

Changes applied to resolve the `>>` operator disambiguation
and remove infix shift operators entirely.
Origin: implementation chat identified parser ambiguity; design
chat ratified the decision and extended it for symmetry.

---

## Design Decision

**No infix shift operators. Both shifts are methods on Int.**

- `>>` is function composition only (precedence level 3)
- `<<` removed as infix operator — no shift level in precedence
- `Int.shift_left(n)` and `Int.shift_right(n)` are the shift API
- Precedence table reduced from 16 levels to 15

### Rationale
- Parser has no type information to disambiguate `>>` at parse time
- Removing `<<` as the lone infix shift eliminates asymmetry
- Composition is a core idiom; bitwise shift is rare in Aura's
  application-level target domain
- `shift_left`/`shift_right` are readable and explicit
- Eliminates an entire precedence level, simplifying the parser

## Spec Changes (aura-spec.md)

### §3.6 — Bitwise operators
**Before:** `**Bitwise:** \`&\`, \`|\`, \`^\`, \`~\`, \`<<\`.`
**After:** `**Bitwise:** \`&\`, \`|\`, \`^\`, \`~\`.`

### §3.7 — Precedence table
**Before:** 16 levels, level 11 was Shift (`<<`)
**After:** 15 levels, Shift level removed entirely, levels 12-16
renumbered to 11-15

### §3.7 — Disambiguation note (replaced)
**Before:** Blockquote about `>>` being composition, `.shr(n)` for
right-shift, `.shl(n)` as alias for `<<`
**After:** Blockquote: "There are no infix shift operators. Use
`Int.shift_left(n)` and `Int.shift_right(n)` methods. `>>` is
reserved exclusively for function composition (level 3)."

### §21.9 — Grammar productions
**Before:**
```
bitwise_and_expr = shift_expr { '&' shift_expr } ;
shift_expr = additive_expr { '<<' additive_expr } ;
```
**After:**
```
bitwise_and_expr = additive_expr { '&' additive_expr } ;
```
`shift_expr` production removed entirely.

## Implementation Implications

- `<<` is no longer a token the lexer needs to emit (unless kept
  for future use — implementation can decide)
- No shift-related handling in expression parser precedence levels
- Precedence climbing now has 15 levels instead of 16
- `Int.shift_left(n)` and `Int.shift_right(n)` registered in
  aura-core (Phase 6, P6.1)
