# Spec Changes — 2026-04-20 17:00 UTC

Language renamed from **Aura** to **BOCK**. AIR expansion redefined
as *Annotated Intermediate Representation*.

---

## Background

"Aura" was a codename chosen for validation speed, not long-term
viability. It had blocking collisions with the Salesforce Aura
framework, UPenn's AURA programming language, Arch Linux's Aura
package manager, and an esoteric Aura language. After validation
succeeded and the language reached usable status, the codename
needed to be resolved before any public artifacts.

The name **BOCK** was selected from a 30+ candidate evaluation
against seven criteria (length, TLD availability, PL/framework/AI
competition, non-tech mindspace, phonetic/visual confusability,
story/solidity, unambiguous pronunciation). The name validates
clean across crates.io (zero collisions), has no competing
programming languages or AI products, has unambiguous US English
pronunciation (`/bɑːk/`), and clean Class 9 trademark landscape.

The BOCK backronym (*Bound Objective Context Kit*) exists for
branding and documentation only. It does not affect technical
terminology in the spec. The spec's existing use of "context"
(context annotations, context composition, context system) remains
the primary meaning and is unaffected by the branding expansion.

## AIR Expansion

**AIR** is now defined as *Annotated Intermediate Representation*
(previously "Aura Intermediate Representation").

Rationale: "Annotated" is the single most accurate descriptor of
AIR's defining characteristic. Every AIR node is a source AST
node plus resolved semantic annotations — type, ownership,
effects, capabilities, context, target info. The layered
progression (S-AIR → T-AIR → C-AIR → TR-AIR) is literally the
process of adding more annotations to each node. "Annotated"
names this directly and distinguishes AIR from other IRs (LLVM
IR's SSA form, MLIR's dialects, Rust's HIR/MIR desugaring stages)
at a glance.
