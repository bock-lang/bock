## FD-01 — `else` on a New Line: APPROVED

**Decision:** Add continuation rule 8: "The next line starts
with `else`."

Both `} else {` and `}\n else {` are valid. The formatter can
enforce a canonical style. `else if` is handled naturally since
`else` triggers continuation and the parser then sees `if`.

## FD-02 — Associated Functions: APPROVED

**Decision:** `Type.method()` syntax for associated functions
(methods without `self`).

Type names are valid in expression position only when followed
by `.method()` (associated function call) or `{ ... }` (record
construction). Type names are not values on their own.

## FD-05 — Numeric Type Coercion: APPROVED

**Decision:** Explicit conversion methods. No implicit coercion.

- `Int.to_float()` → `Float`
- `Float.to_int()` → `Int` (truncates toward zero)
- `to_string()` on all primitives

Truncation direction is explicitly toward zero (not floor),
matching Rust's `as` cast and JavaScript's `Math.trunc()`.
