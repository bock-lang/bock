## s09-context.md — EDITED

### Context Inheritance Rules section added
**What:** New section after `Context in AIR` specifying override
vs additive inheritance behavior for context annotations.
**Why:** P8.3 (Context Composition, rewritten in design review)
specifies that declaration-level annotations override module-level
for most annotation types, except `@requires` which is additive
(union). The spec excerpt only said "inherited through tree" with
no detail on override semantics. Without this fix, a Claude Code
session on P8.3 would see the package file's rules conflicting
with the spec excerpt's vague "inherited" language.

**Rules added:**
- Declaration-level overrides module-level (same annotation kind)
- `@requires` is additive (declaration capabilities ∪ module
  capabilities)
