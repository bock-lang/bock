## Decision: `>>` is composition only

`>>` is always function composition (precedence level 3).
Bitwise right-shift uses the `.shr(n)` method. `.shl(n)` is
available as a method alias for `<<` for symmetry.

### Rationale
- Parser has no type info to disambiguate by operand type
- Composition is a core idiom; bitwise shift is rare in Aura's
  application-level target domain
- `.shr()` is consistent with Kotlin and Haskell conventions
- Eliminates the ambiguity entirely with no special cases

### Implications for Implementation (F2.05)
- `>>` token stays at precedence level 3 (compose) — no change
- Remove `>>` from precedence level 11 shift handling
- `<<` remains as the sole infix shift operator at level 11
- Register `Int.shr(n)` and `Int.shl(n)` in aura-core
- `BinOp::Shr` in AST can remain but is never produced by parser
