# Spec Changes — 2026-04-04 14:00

Addition of §17.8 — AI Provider Interface and updated Appendix A
`[ai]` configuration section.

---

## Context

The spec (§17) defines an AI-first transpilation pipeline but was
silent on how the compiler connects to AI models. Implementation
requires a concrete provider interface before Phase 9 (codegen)
can support Tier 1 generation.

The implementation chat proposed a pluggable two-provider
architecture. The design chat reviewed it and made five
corrections before adoption:

1. **No vendor names in the spec.** Provider types, model names,
   and endpoint URLs belong in the tooling guide, not the spec.
   The spec defines the interface contract and configuration
   schema only.

2. **Three interaction modes, not one.** The trait needs to
   support Generate (Tier 1), Repair (§17.7 feedback loop),
   and Optimize (Tier 3) — not just a single `synthesize()`.

3. **Verification removed from provider trait.** Verification
   (§17.3) is always deterministic and doesn't involve AI.
   It belongs on the target profile, not the AI provider.

4. **Caching semantics specified.** Content-addressed cache
   keyed on request hash. Cached responses are treated as
   pinned decisions for reproducibility.

5. **Placed as §17.8, not §17.2.1.** The pipeline section
   (§17.1–§17.4) describes the WHAT. The provider interface
   is HOW and belongs in its own subsection.

---

## Spec Amendments

### §17.8 — AI Provider Interface — NEW
Inserted after §17.7 (Codegen Rule Learning). Covers:
- Provider-agnostic interface concept
- Three interaction modes (Generate, Repair, Optimize)
- Verification is separate (not on the provider)
- Two built-in providers (OpenAI-compatible, Anthropic native)
- Configuration via `[ai]` section
- Content-addressed caching with pinned-decision semantics

### Appendix A — `[ai]` section — REPLACED
**Before:**
```toml
[ai]
model = "aura-codegen-v3"
deterministic_fallback = true
auto_pin = false
```

**After:**
```toml
[ai]
provider = "openai-compatible"
endpoint = "https://api.example.com/v1"
model = "model-name"
api_key_env = "AI_API_KEY"
deterministic_fallback = true
auto_pin = false
cache = true
max_retries = 3
timeout_seconds = 30
```

Uses placeholder values (not vendor-specific). `api_key_env`
stores the env var NAME, never the key itself.

### playbook/specs/s13-transpilation.md — EDITED
Added "AI Provider Interface" section summarizing the three
interaction modes, verification separation, and two built-in
providers.

## Implementation Implications

This is post-v1 infrastructure. The current Phase 9 packages
(P9.1–P9.6) implement Tier 2 (rule-based) codegen only. The
provider interface will be needed when Tier 1 is implemented,
likely as a Phase 11 or post-v1 extension alongside the §17.7
feedback loop.

No changes to the current 61-package plan.
